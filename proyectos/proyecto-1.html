¿Qué es este proyecto?

Sistema de información desarrollado para la Escuela Militar de Transmisiones, diseñado para administrar el proceso de ingreso, seguimiento y control de cadetes. La aplicación centraliza los registros de la institución, evita duplicidades y permite consultar el historial de cada cadete de forma rápida y ordenada.

Función principal

El sistema gestiona el registro, alta, baja y consulta de cadetes ingresados a la institución. Permite capturar datos personales e información militar de cada cadete, así como llevar un control de su estado dentro de la escuela. Todo el flujo opera desde una interfaz web interna que se conecta directamente con la base de datos institucional mediante Hibernate como capa de persistencia.

Stack tecnológico

HB Hibernate ORM — persistencia de datos
JSF JavaServer Faces Framework — capa de vista
DB MariaDB SQL Base de datos relacional
☕ Java EE Lógica de negocio
NB NetBeans 8.2 IDE de desarrollo

Lo que aprendí

Este proyecto fue mi primer acercamiento real a un sistema con persistencia de datos usando Hibernate como ORM. Aprendí a mapear entidades Java con tablas relacionales, a manejar sesiones y transacciones, y a construir una interfaz funcional con JSF conectada a MariaDB. También reforcé buenas prácticas de organización del proyecto dentro de NetBeans y el flujo de trabajo con un cliente institucional real.
